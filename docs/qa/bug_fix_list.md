# 修复单（Bug Fix List）

本文档按编号整理 QA 人工测试及后续发现的遗留问题，便于开发排期与闭环。编号规则：**B{开发阶段}-{开发号}-{Bug序号}**（如 B-P1-1-001）。

---

## 编号规则


| 部分    | 说明                          | 示例  |
| ----- | --------------------------- | --- |
| B     | Bug 标识                      | B   |
| 开发阶段  | P0 / P1 / P2 / P3 / P4 / P5 | P1  |
| 开发号   | 该阶段内批次或模块序号（从 1 起）          | 1   |
| Bug序号 | 该批次内流水号，三位数字                | 001 |


**示例**：`B-P1-1-002` 表示 P1 阶段第 1 批第 2 个 Bug。

---

## 修复单列表

### B-P1-1-001


| 项         | 内容                                                                                            |
| --------- | --------------------------------------------------------------------------------------------- |
| **关联测试项** | [app_manual_test_checklist](app_manual_test_checklist.md) P1-11                               |
| **标题**    | d100（2d10）结果应按十位+个位展示，而非两数相加                                                                  |
| **现象描述**  | 使用 d100（两颗 d10）掷骰时，当前将两颗骰子点数相加展示（如 3 与 7 显示为 10）。按 d100 语义应为十位与个位组合，即 3 和 7 应表示 37，而非 3+7=10。 |
| **复现步骤**  | 工具 → 骰子 → 选择或输入 d100（2d10）→ 掷骰 → 查看结果展示方式。                                                    |
| **预期**    | 两颗 d10 分别作为十位、个位，结果展示为 1～100 的数值（如 07、37、100）；或产品明确采纳「相加」规则并在文案中说明。                           |
| **严重程度**  | 中                                                                                             |
| **状态**    | 待验证                                                                                           |
| **来源**    | 2026-02-23 人工测试                                                                               |


---

### B-P1-1-002


| 项         | 内容                                                                        |
| --------- | ------------------------------------------------------------------------- |
| **关联测试项** | [app_manual_test_checklist](app_manual_test_checklist.md) P1-12           |
| **标题**    | 未确认舍弃/重 Roll 时再次掷骰，上次结果应记为舍弃并进入历史                                         |
| **现象描述**  | 用户掷骰后，在未点击「舍弃」或「重 Roll」的情况下再次掷骰，当前行为未明确。期望：上次掷骰结果视为舍弃，并进入历史记录，当前展示为本次新结果。 |
| **复现步骤**  | 工具 → 骰子 → 掷 d6 → 不点舍弃/重 Roll → 再掷 d6 → 观察历史与当前展示。                         |
| **预期**    | 再次掷骰时，上一轮结果自动记为舍弃并写入历史；当前仅展示本轮结果。                                         |
| **严重程度**  | 低                                                                         |
| **状态**    | 待验证                                                                       |
| **来源**    | 2026-02-23 人工测试                                                           |

### B-P1-1-003


| 项         | 内容                                                              |
| --------- | --------------------------------------------------------------- |
| **关联测试项** | [app_manual_test_checklist](app_manual_test_checklist.md) P1-15 |
| **标题**    | 诗签展示缺少签头文案，直接显示签 id 不雅观                                         |
| **现象描述**  | 诗签抽签后展示签文时，缺少人性化签头（如「妈祖灵签 第 11 签」），当前直接显示签 id，体验不佳。             |
| **复现步骤**  | 工具 → 诗签 → 抽签 → 查看签条标题/签头区域。                                     |
| **预期**    | 展示签头文案，如「妈祖灵签 第 11 签」或与库名、签号对应的 l10n 文案，不直接裸露签 id。              |
| **严重程度**  | 低                                                               |
| **状态**    | 待验证                                                             |
| **来源**    | 2026-02-23 人工测试                                                 |
